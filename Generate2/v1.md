# Generate2 v1 — public API

## Embedding contract

Generate2 is a SwiftPM library embedded by a parent app. The parent owns the
`NavigationStack`. Generate2 owns no routes and presents no parent-domain
sheets (topup, song picker, team derive view).

## ContentView

Two modes via two public inits:

### Normal mode

```swift
ContentView(
    tokenid: String,
    onTopupNeeded: @escaping () async -> Bool,
    onImageTapped: @escaping (String) -> Void,
    onUploadSong: @escaping () -> Void
)
```

### Derive mode

```swift
ContentView(
    tokenid: String,
    onTopupNeeded: @escaping () async -> Bool,
    onImageTapped: @escaping (String) -> Void,
    onUploadSong: @escaping () -> Void,
    filename: String,
    tweak: String
)
```

Parent pushes this onto its stack after its team's "Change it" view collects
a `tweak` for `filename`. Generate2 fires the derive pipeline on appear.

## Closures

| Closure          | Fires when                                  | Parent does                                     |
| ---------------- | ------------------------------------------- | ----------------------------------------------- |
| `onTopupNeeded`  | Credit gate runs dry mid-flow               | Show purchase UI; resolve `true` on success     |
| `onImageTapped`  | User taps an image in the grid              | Push team's "Change it" view with the path      |
| `onUploadSong`   | User taps the song-title slot in the toolbar | Present song-picker sheet                      |

## `onImageTapped` payload

Single arg: full on-disk path as `String`.

- Server filename: `URL(fileURLWithPath: path).lastPathComponent`
- Bytes: `Data(contentsOf: URL(fileURLWithPath: path))`

Rationale: parent can read the file without knowing Generate2's storage
layout (`Documents/<project>/<filename>`), and still recovers the server
filename via `lastPathComponent` for any API call on the next screen.

## What Generate2 owns

- API calls (auth, pricing, generation, balance)
- Local project storage (`ProjectService`)
- Image grid + chrome

## What the parent owns

- `NavigationStack` and all routes
- Topup / purchase UI (StoreKit)
- Song picker sheet
- Team's derive ("Change it") view
